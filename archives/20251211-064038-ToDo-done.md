# ToDo.md

## **重要事項**
- このファイルは、ユーザーが編集します。AIが編集することを禁止します。
- コメントは日本語で、一行の長さは78文字以内にしてください。
- git操作はユーザーが行います。AIはgit操作を行わないでください。


## ToDo項目

- [ ] 上記「重要事項」を確認してください。疑問があればユーザーに質問してください。

- [ ] tests/ 以下のテストプログラムをリファクタリングしたい。テストプログラムのコードを検証して、リファクタリング計画を策定し、``Plan.md``に日本語でまとめてください。
- [ ] Plan.mdができたら、ユーザーに確認を求めてください。次の作業には勝手に進まないでください。

- [ ] Plan.mdに基づいて、具体的なタスクをステップバイステップで検討して、タスクリストをTask.mdに保存して。
- [ ] Task.mdに従って、順番にタスクを実行してください。タスク項目が完了したら、その項目にチェックを入れてから、次のタスク項目を実行してください。
- [ ] すべてのタスクが完了したら、ユーザーに報告してください。ユーザーの指示があるまで、勝手に次に進まないでください。

- [ ] 次のコマンドで、Task.mdをアーカイブする。``uv run archivefile Task.md``
- [ ] ``mise run test``を実行して、問題がないか確認する。
- [ ] 完了したら、ユーザーに報告する。ユーザーの指示があるまで、勝手に次に進まないでください。
